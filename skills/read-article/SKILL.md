# 記事を一緒に読む

記事やスレッドの内容を、ユーザーの現状の仕組み・知識と照合して仕訳する。

## 引数

`$ARGUMENTS` に記事テキスト、URLを指定する。

引数が空の場合は以下を表示して終了する:
> 記事のテキストを貼り付けてください。X のスレッドなら Grok にまとめてもらったものでも OK です。

## 手順

1. 記事の内容を読み取る（URLが渡された場合はfetchを試みる。取得できない場合はテキスト貼り付けを依頼する）
2. 以下の情報源を確認し、ユーザーの現状を把握する:
   - グローバル `CLAUDE.md` やプロジェクト `CLAUDE.md`
   - `MEMORY.md` とメモリファイル
   - `plans/` 配下のプラン（あれば）
   - 関連しそうなオープンIssue（`.issue-index.json` を `Read`。ファイルがなければ `bash ~/workspace/github.com/mikkegt/claude/skills/issue-index/sync.sh .` で生成）
3. 記事の内容を以下の3カテゴリに仕訳して報告する

## 出力フォーマット

### 仕訳結果

**既にやっていること**
- （記事の内容）→（ユーザーの該当する仕組み・設定）

**新しい情報・取り入れられそうなこと**
- （内容）→（具体的にどう活かせるか）

**関連するIssue・プラン**
- #XX（Issue名）— どう関連するか

### 一次情報チェック

- この記事は一次情報か、二次情報（まとめ・感想）か
- 一次情報がある場合、そのソースを提示

### 煽りワード検出

記事が二次情報・煽り記事と判定された場合のみ表示:
- 検出した煽りフレーズを列挙
- `lists/x-mute-words.md`（Jarvisリポジトリ）に未登録のものがあれば追加を提案

### 総合判断

「読む価値があったか」「アクション不要か」を一言で。

## ルール

- 忖度しない。既にやっていることは「既にやっている」と言い切る
- バズ記事の煽りに乗らない。冷静に判定する
- 「すごい」「参考になる」等の感想は不要。事実ベースで仕訳する
- 保存が必要な場合は `content/clippings/YYYYMMDD-{slug}.md` への保存を提案する（外部記事は必ず `clippings/` 配下に保存し、自分の記事と混ぜない）
- 保存時は frontmatter に `source` / `author` / `published` / `saved` / `type: clipping` を入れる
