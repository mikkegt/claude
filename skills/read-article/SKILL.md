# 記事を一緒に読む

記事やスレッドの内容を、ユーザーの現状の仕組み・知識と照合して仕訳する。

## 引数

`$ARGUMENTS` に記事テキスト、URLを指定する。

引数が空の場合は以下を表示して終了する:
> 記事のテキストを貼り付けてください。X のスレッドなら Grok にまとめてもらったものでも OK です。

## 手順

1. 記事の内容を読み取る（URLが渡された場合はfetchを試みる。取得できない場合はテキスト貼り付けを依頼する）
2. 以下の情報源を確認し、ユーザーの現状を把握する:
   - グローバル `CLAUDE.md` やプロジェクト `CLAUDE.md`
   - `MEMORY.md` とメモリファイル
   - `plans/` 配下のプラン（あれば）
   - 関連しそうなオープンIssue（`.issue-index.json` を `Read`。ファイルがなければ `bash ~/workspace/github.com/mikkegt/claude/skills/issue-index/sync.sh .` で生成）
3. 記事の内容を以下の3カテゴリに仕訳して報告する

## 出力フォーマット

### 仕訳結果

**既にやっていること**
- （記事の内容）→（ユーザーの該当する仕組み・設定）

**新しい情報・取り入れられそうなこと**
- （内容）→（具体的にどう活かせるか）

**関連するIssue・プラン**
- #XX（Issue名）— どう関連するか

### 一次情報チェック

- この記事は一次情報か、二次情報（まとめ・感想）か
- 一次情報がある場合、そのソースを提示

### 煽りワード検出

記事が二次情報・煽り記事と判定された場合のみ表示:
- 検出した煽りフレーズを列挙
- `lists/x-mute-words.md`（Jarvisリポジトリ）に未登録のものがあれば追加を提案

### 総合判断

「読む価値があったか」「アクション不要か」を一言で。

## ルール

- 忖度しない。既にやっていることは「既にやっている」と言い切る
- バズ記事の煽りに乗らない。冷静に判定する
- 「すごい」「参考になる」等の感想は不要。事実ベースで仕訳する
- 仕訳した記事は `content/clippings/INDEX.md` の表に1行追加する（デフォルトstatus: `read`）
- **本文は基本的に保存しない**。INDEXに出典リンクとメタ情報のみ記録する
- 本文を保存する条件は以下のいずれか:
  - X投稿・スレッドなど消えやすいソース
  - ユーザーが明示的に「全文残して」と指示した
  - その場合 `content/clippings/archive/YYYYMMDD-{slug}.md` に保存し、frontmatter（`source` / `author` / `published` / `saved` / `type: clipping`）必須。INDEX.mdの出典欄に `／ [archive](archive/...)` リンクを追加
- 記事内容が実装/採用されたらINDEXのstatusを `implemented` に更新し、関連欄に成果物リンクを追加
- 詳細な運用ルールは `content/clippings/INDEX.md` の「運用ルール」セクション参照
