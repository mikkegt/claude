Pull Request #$0 をレビューしてください。

## 手順

1. PRのdiffを取得: `gh pr diff $0`
2. PRの説明を取得: `gh pr view $0`
3. 変更されたファイルを理解し、以下の5軸でレビュー

## レビュー5軸

各項目を確認し、判定結果を表にまとめる。

| 軸 | 判定 | 所見 |
|----|------|------|
| セキュリティ | ✅ or ⚠️ | インジェクション、認証漏れ、秘密情報の露出 |
| パフォーマンス | ✅ or ⚠️ | N+1クエリ、不要な再レンダリング、重い処理 |
| 保守性 | ✅ or ⚠️ | 6ヶ月後の自分が読めるか、命名、構造 |
| エッジケース | ✅ or ⚠️ | null、空配列、並行処理、境界値 |
| 過剰設計 | ✅ or ⚠️ | YAGNI違反、不要な抽象化、使われないオプション |

## 出力フォーマット

1. **概要**: PRが何をしているか1-2文で
2. **判定表**: 上記の5軸テーブル（⚠️の項目は具体的な箇所と改善案を記載）
3. **Good**: 良い点があれば簡潔に
4. **総合判定**: LGTM / 要修正
