# AI くささ除去

対象ファイルから AI っぽい表現を検出・修正し、自然な文体にする。元の意味や情報は変えない。

## 引数

`$ARGUMENTS` に対象ファイルのパスを指定する（必須）。

引数が空の場合は以下を表示して終了する:
> 対象ファイルのパスを指定してください。例: `/humanize .claude/misato/docs/report-20260404-weekly-notion.md`

## 手順

1. `$ARGUMENTS` のファイルを読み込み、下記チェックリストで AI くさい箇所を検出する
2. 検出結果を一覧で表示する（行番号、該当箇所、問題点、修正案）
3. 「修正版を作成しますか？」とユーザーに確認する
4. OK が出たら修正後の全文を提示し、セルフ監査パスを実行する
5. 検出ゼロの場合は「問題ありませんでした」と表示して終了する

## チェックリスト

### 語彙・表現

**AI 頻出語彙を置き換える**

| NG | 代替例 |
|-----|--------|
| 浮き彫りにする | 分かった、見えた |
| 〜を示唆しています | 〜と読める、〜かもしれない |
| 〜において | 〜で |
| 〜を鑑みると | 〜を考えると |
| 包括的な / 網羅的な | 全体の / 幅広い |
| 〜に寄与する | 〜に役立つ、〜につながる |
| 〜として位置づけられる | 〜である |
| 〜の役割を果たす | 〜になっている |
| 特筆すべき | 目立つ、大きい |
| 多岐にわたる | いろいろな |
| 〜することが可能です | 〜できます |
| 〜を行う | 〜する |
| 〜に関して | 〜について |
| 〜という形になります | 〜です |
| 〜の観点から | （多くの場合、削除可能） |
| 浮上する | 上に来る、上がる |
| 出現する | 入った、出てきた |
| 合意する / 合意を得る | 決める、決まる、OK をもらう、まとまる、すり合わせる |
| 合意した | 決めた、まとまった |
| 合意形成 | すり合わせ、調整 |
| 合意事項 | 決めるべきこと、すり合わせポイント |
| 方向性合意 | 方向性の確認、方針の確認 |
| 踏襲する / 踏襲 | 真似ている、引き継いでいる |
| 模倣する / 模倣 | 真似ている |
| 観察 (体言止め、または「〜を観察」) | 見えた、出た、確認した |
| 推測される | と思われる、とみる |
| 留意する | 気をつける |
| 副次的に / 副次的な | あわせて、もう一つの |
| 本検証 / 本検証群 | 今回の検証 |
| 別途 | 別件で、別に |
| 概ね | ほとんど |

**漢語熟語の機械的多用に注意**: 「踏襲」「模倣」「観察」「推測」「留意」「副次的」「本検証」「別途」「概ね」のような漢語熟語が短い節に密集していたら、複数を口語に置き換える。特に「踏襲も観察」「観察された」「副次的に〜本検証では」のように体言止めや改まった節と組み合わさるとレポート風が強まる。

**接続詞の連発を避ける**: 「さらに」「加えて」「また」「一方で」が連続していたら、一部を削除するか文をつなげる。

**同義語の水増しをやめる**: 「検討・考察・分析」のように同じことを言い換えて並べない。

**過剰なヘッジングを減らす**: 「〜かもしれません」「〜の可能性があります」「〜と考えられます」の連発を避け、言い切れるところは言い切る。

### 構造パターン

- 箇条書きの項目名を太字にしない（`**速度:** 大幅に改善` → `速度: 大幅に改善`）
- 太字を機械的に多用しない。本当に強調したい箇所だけ
- ダッシュ（——）を括弧や読点で代替する
- 「重要です」「不可欠です」等の意義の過剰強調を削る
- 「研究によると」等、具体的な出典がない言及を削る
- 体言止めを多用しない。文末が連続して名詞で終わると箇条書きの羅列っぽくなる
- 単発でも、述語の省略で動作なのか状態なのか判別しづらい体言止めは避ける
  - NG: 「次回のタスクの確認。」（確認した? 確認する?）
  - OK: 「次回のタスクを確認する。」

### 冒頭と結論

- 「ここでは〜について報告します」で始めない。いきなり本題に入る
- 「今後の展開が期待されます」「注目されます」で終わらない。具体的に何をするか書く

### トーン

- 簡潔、事実ベース、だが無味乾燥すぎない
- 社内報告書として自然な硬さ（ビジネスメール程度）
- 読み手が知りたいことを先に書く（結論ファースト）
- 「承知しました」「素晴らしい質問ですね」等のチャットボット表現を消す
- 敬体（〜です、〜ます）と常体（〜だ、〜である、〜する）を混在させない。文書全体でどちらかに統一する

## セルフ監査パス

修正後に以下を確認し、該当があれば再修正する:

1. 冒頭が定型パターン（「ここでは〜」）になっていないか
2. 結論が定型パターン（「期待されます」「注目されます」）になっていないか
3. 同じ接続詞が近距離で 3 回以上出ていないか
4. 不要な太字が 5 個以上ないか
5. 全体を通して「人間が書いた社内メールっぽい自然さ」があるか
6. 体言止めの文末が近距離で 3 つ以上連続していないか
7. 単発でも意味があいまいな体言止めが残っていないか
8. 敬体と常体が混在していないか
