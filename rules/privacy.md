# プライベート→パブリックの持ち込み防止ルール

事故が起きるのは「プライベートリポジトリで作業中に、その文脈を持ったまま
パブリックリポジトリ（公開しているグローバル設定・ルール等）へ書き込むとき」。
パブリックだけ／プライベートだけを触る作業では起きない。この遷移だけを警戒する。

## 発動条件

- プライベートリポジトリで作業中に、パブリックリポジトリへファイルを作る／更新するとき
  （典型: 作業中に得た知見を公開ルール / CLAUDE.md に追記する場面）

## そのとき確認すること

- MUST 書く内容にプライベート由来の固有情報が紛れていないか見直してから書く:
  案件名・プロダクト名・コード名・リポジトリ名 / 未公開の設計・数値・体制・取引先 /
  個人名（無名化）/ ローカル限定パス / ユーザーの生の指示文（User Prompt）
- MUST 例や背景は固有名詞を伏せて一般化する（「あるプロジェクトで」等）
- MUST 迷ったら本人に確認する

## なぜ

公開リポジトリは一度書くとログ・履歴・インデックスに残り、後から消しても辿れる。
プライベート→パブリックの遷移が唯一の危険地帯なので、そこだけ確実に止める。
