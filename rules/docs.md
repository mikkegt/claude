# ドキュメント整理ルール

`.claude/<user>/docs/` などに**個人的なレポート・メモ**が溜まる場合のルール。
プロジェクト本体のコードドキュメント (README, docs/) には適用しない。

## 命名規則

```
<type>-YYYYMMDD-<slug>.md
```

| type | 用途 |
|---|---|
| `report-` | 検証 / 週次 / 設計の正式レポート (外部提出 or 後で参照する対象) |
| `mtg-` | MTG 議事録 (`mtg-report-YYYYMMDD-*` でも可) |
| `note-` | 個人備忘メモ |
| `slack-` | Slack 投稿ドラフト / 共有用 |
| `todo-` | TODO リスト |
| `current-` | 現在進行中のタスク (固定名 `current-task.md`) |

YYYYMMDD は作成日。slug はトピックを 1-3 単語の英小文字 + ハイフン区切りで。

## INDEX.md

ファイルが **10 件以上** 溜まったら、ディレクトリ直下に `INDEX.md` を作成する。

- テンプレ: `~/workspace/github.com/mikkegt/claude/templates/INDEX-template.md`
- 役割: **Claude Code が最初に読むファイル**。「過去のあの件どうだったっけ?」と聞かれた時に、INDEX → 該当ファイルへ 1 ジャンプできる構造にする
- メンテ: 手動 (40 ファイル未満なら現実的)

## archive ルール

以下のファイルは `archive/` サブディレクトリへ **移動** する (削除しない):

- **3 ヶ月以上古い** (現在の作業と関連が薄い)
- **重複ファイル** (例: `-MTG用.md`, `-draft.md` などの派生版)
- **集約レポートに統合された個別レポート** (集約版だけ残す)
- **週次レポート系で時系列で参照しないもの**

**迷ったら archive** (削除より安全、後から戻せる)。

## メンテのタイミング

- ファイル数が **20+** になったら棚卸し + archive 化を検討
- 新規レポート追加時に INDEX.md の「トピック別索引」に追記
- 半年ごとに INDEX.md 自体を見直す (機能不全なら自動生成 Skill 化を検討)
