# ライティング（人間向け出力）

ユーザーへの回答・レポート・人間向け docs に常時適用する軽い既定。
深く整える時は humanize skill を併用する（AI 臭の重い清掃はそちら）。

- MUST humanize skill はユーザーが明示的に依頼したときだけ起動する。PR 本文・
  コミットメッセージ・コード作業などの中で自発的に起動しない。この writing.md の
  記述は「humanize という skill がある」という参照であって、自発起動の指示ではない

- SHOULD bold を多用しない。本当に強調したい箇所だけ。箇条書きの項目名は
  太字にしない（`速度: 改善` でよい）
- SHOULD 過剰なヘッジングを避ける。言い切れるところは言い切る
- SHOULD 「重要です」「不可欠です」等の過剰な強調表現を削る
- Claude が読む設定ファイル（rules/ 等）では bold は実効が無いので使わない
