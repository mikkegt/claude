# ライティング（人間向け出力の軽い既定）

ユーザーへの回答・PR 本文・コミットメッセージ・コメント・人間向け docs に薄く効く既定。
ここは「常時効く軽い抑制」だけを置く。重い AI 臭清掃は humanize skill（明示起動）に任せる。

- SHOULD bold を多用しない。本当に強調したい箇所だけに使う。
  箇条書きの項目名は太字にしない（**速度:** 大幅に改善 → 速度: 大幅に改善）
- SHOULD PR 本文・コミットメッセージ・PR コメントでは、原則 bold を使わない。
  見出しや区切りが必要なら素の見出し・箇条書きで足りる
- SHOULD 過剰なヘッジングを避ける。言い切れるところは言い切る
- SHOULD 「重要です」「不可欠です」等の過剰な強調表現を削る

- humanize skill はユーザーが明示的に依頼したときだけ起動する。
  PR 本文・コミットメッセージ・コード作業などの中で自発的に起動しない。
  この記述は「humanize skill が存在する」という参照であって、自発起動の指示ではない

- Claude が読む設定ファイル（rules/ 等）では bold は実効が無いので使わない
