# Claude Code が気を効かせて振ってくる識別子の命名ルール

Claude Code は頼まれなくても連番・記号・略語（T1, Phase A, 案2 等）を
気を効かせて振ってくることがある。これが人間/チームの語彙と混ざると、
日が経つと「この記号は誰が何の略で振った?」が分からなくなる。

## ルール

- MUST Claude Code が独自に振る識別子（連番・フェーズ記号・独自略語）には
  接頭辞 `CC-` を付ける（例: `CC-1`, `CC-2`, `CC-PhaseA`）
- NEVER 人間/チームが既に使っている記号体系に勝手に項目を足さない。足す必要が
  あれば確認し、足したものには出所が分かる注記を残す
- MUST 連番・略語の意味（何の略か）は、その体系を作った人が初出箇所に 1 回定義する
- 発動条件: 永続ドキュメント・レポートに残す識別子のみ。会話内の使い捨ては不要

## なぜ命名で区別するのか

凡例（説明文）は付け忘れると出所が消える。`CC-` を命名に埋め込めば、記号を
見た瞬間に Claude 由来か分かり、付け忘れようがない。

## 経緯

人間のドキュメントに Claude が連番を 1 個足したところ、後日「その記号は誰由来か /
何の略か」が分からなくなった。出所が命名になく、定義も残っていなかったため。
`CC-` は制定後（2026-06-15〜）の新規分から適用する。
