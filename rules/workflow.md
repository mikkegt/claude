# 開発ワークフロー

## バグ修正 / 機能要望

1. MUST ユーザーと要件を議論・明確化する
2. MUST スコープが明確になったらGitHub Issueを作成する
3. MUST リポジトリの `plans/` ディレクトリにプランファイルを作成する
4. MUST プランに基づいて実装する
5. MUST PR/MRの説明に「背景」セクション（なぜこの変更が必要か）を含める。ユーザーの生のプロンプトをそのまま転記しない。Claude Code の署名（🤖 Generated with Claude Code）は含めない

## コミット / PR の標準フロー

1. MUST コミット前にコミットメッセージを提案し、承認を得てからコミット & プッシュ
2. MUST プッシュ後に PR 本文を提案し、承認を得てから PR 作成（PR 作成は外部発信なので原則ユーザーが行う）

## ドキュメントのメンテナンス

- MUST 変更後にREADME.mdを確認し、正しい仕様を反映する
- MUST コマンド例、オプション、使い方の説明が正確であることを確認する
